# 口袋旅行 Pocket Travel

一个手机优先的旅行规划小应用。首页列出你的每一趟旅行（东京、香港…），点进去是一张真实地图 +
逐日行程 + 可拖拽的「随缘泡泡」（附近备选点）+ 语言急救包 + 打卡记录。为「怕把旅行过成打卡任务」
而设计：只有少数几个硬性锚点（📌），其余都是菜单不是任务。行程、随缘点、锚点、语言包、情报页
都能在应用里直接增改删——机票改签、酒店换了，点开卡片 ✎ 编辑即可（锚点的改与删都会二次确认）。
云端共享时，还能看到**此刻同在看这份行程的同行人**。

## 两种模式

不配置任何东西，直接部署就能用（**本地模式**）。想要账号和好友共享时，接上一个免费后端即可切到
**云端模式**——两种模式界面一样，云端是本地的超集。

| 模式                 | 怎么触发                                      | 有什么                                                                                       |
| -------------------- | --------------------------------------------- | -------------------------------------------------------------------------------------------- |
| **本地模式**（默认） | `app/config.js` 留空                          | 多个行程、导入/导出、离线可用、全部行程功能。数据存本机浏览器，无需登录。                    |
| **云端模式**         | 在 `app/config.js` 填 Supabase URL + anon key | 以上 + 邮箱/谷歌登录、行程存云端（换设备也在）、把行程分享给朋友后**双方实时同步一起编辑**、看到**正在同看这份行程的同行人**。 |

启用云端与共享的详细步骤见 **[SETUP.md](SETUP.md)**（约 15 分钟）。

## 快速开始

**在线用：** 部署到 GitHub Pages（仓库 Settings → Pages → 选 main 分支根目录），打开首页即可。
手机上「添加到主屏幕」就是一个全屏小 App，地铁里没信号也能用（地图瓦片除外）。

**本地跑：** 因为拆成了多文件，建议用本地服务器打开（直接双击 `trip.html` 在部分浏览器会因为
跨文件加载被拦）：

```bash
python3 -m http.server 8000
# 浏览器打开 http://localhost:8000/
```

## 目录结构

```
index.html          首页：行程列表、登录（云端模式）、导入/导出、共享
trip.html           行程规划器本体（地图 + 行程 + 泡泡 + 语言包），以 trip.html?trip=<id> 打开
app/config.js       唯一开关：Supabase URL/key（留空=本地）、登录方式、App 名字
app/store.js        数据层 window.PT：本地/云端两套后端，同一套接口
vendor/supabase.js  本地打包的 Supabase 客户端（不走 CDN）
db/schema.sql       Supabase 建表 + 权限 + 共享函数（在 SQL Editor 跑一次）
SETUP.md            开启云端 & 共享的说明
CLAUDE.md           给 AI/开发者的代码地图
```

## 技术

无构建、无框架的静态站点。地图用 Leaflet + CartoDB Voyager 瓦片（Google 官方地图需要 API key
和计费账户，塞不进纯静态单文件，这是最接近的免费方案）。云端后端用 Supabase（开源的 Firebase
替代：Postgres + 行级权限 + 实时）。**只有地图瓦片需要联网**，行程数据、泡泡、拖拽、语言包全部离线。语言急救包不再写死在页面里，
而是行程数据的一部分（可在应用内按分组增删句子；清空即隐藏该 tab）。

## 数据安全

云端模式下，行程数据的访问由数据库行级权限（RLS）保证：没登录看不到任何行程；登录了也只能看到
「自己的 + 别人分享给你的」。因此 `config.js` 里的 **anon key 可以放心提交到公开仓库**——它本就是
给浏览器用的公开值。（唯一不能提交的是 Supabase 里的 `service_role` key，本项目也用不到它。）
